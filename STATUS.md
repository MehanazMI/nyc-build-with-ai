## 🚨 Important Notes

### API Limits
- Free tier: 15 RPM, 1500 RPD
- During testing: Use fewer calls
- Cache results when possible
- During hackathon: You'll get credits!

### Focus Areas
- Make 1-2 features **work really well**
- Better than 5 half-working features
- Demo quality > code perfection
- Story matters (problem → solution)

### Common Mistakes to Avoid
- ❌ Too many features
- ❌ No clear use case
- ❌ Over-engineering
- ❌ Poor error handling
- ❌ No backup demo plan

### Success Formula
- ✅ Clear problem statement
- ✅ 1-2 agents working smoothly
- ✅ Engaging 3-minute demo
- ✅ Live interaction (with backup)
- ✅ Emotional story

---

## 📞 When Things Go Wrong

### API Errors
- Have pre-recorded responses ready
- Demo with screenshots/video
- Explain what WOULD happen

### Technical Issues
- Always have Plan B
- Test everything twice
- Backup videos are OK!

### Blank Mind During Demo
Memorize this structure:
1. "We built X to solve Y problem"
2. "Let me show you how it works"
3. "Watch as our AI agent..."
4. "This helps users by..."
5. "Thank you!"
